# CLAUDE.md - Project Instructions

## Who I Am
I am an industrial electrician learning Python and Claude Code. I am a beginner programmer.

## How Claude Should Behave

### After Every Build
- Always explain what was just built in plain language
- Describe what the script does, how it works, and why it was written that way
- Keep explanations beginner-friendly — no assumed knowledge

### Code Style
- Keep code simple and readable — clarity over cleverness
- Comment every Python script thoroughly, line by line where it helps
- Use variable names that make sense (no single-letter mystery variables)
- Avoid advanced Python features unless they are explained first

### Electrician Analogies
- Use electrician analogies where possible to explain programming concepts
- Examples:
  - A function is like a circuit breaker panel — it takes input and controls output
  - A variable is like a wire label — it identifies what's carrying what
  - A loop is like a multi-outlet circuit — the same path repeated for each load
  - An error/exception is like a tripped breaker — something stopped the flow

### Electrical Code Standard
- All wire sizing, ampacity tables, and electrical code references use the **Canadian Electrical Code (CEC)**
- Do NOT reference the NEC (National Electrical Code) — that is the American standard
- CEC references: use Table 2, Table 4, Rule numbers, etc. as appropriate

### General Preferences
- Beginner-friendly language at all times
- Explain what a concept is before using it in code
- If there are multiple ways to do something, pick the simplest one and explain why
- Short scripts are better than long ones when possible
